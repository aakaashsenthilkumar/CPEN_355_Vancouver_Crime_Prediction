## Overview

This project predicts a **safety score (0–100)** for Vancouver neighbourhoods using historical crime data from the Vancouver Police Department (VPD) Open Data portal. A weighted scoring system is designed to reflect crime severity, three regression models are trained and compared, and the results are exposed through a CLI query tool, a terminal chatbot, and a full GUI chatbot — all powered by a locally running LLM (free, no API key required).

## Setup

### Prerequisites
- Python 3.9 or higher
- Ollama (only needed for `chatbot.py` and `gui.py`)

### 1. Install Python dependencies
```bash
pip install -r requirements.txt
```

### 2. Add the dataset
Download the VPD crime dataset (All Neighbourhoods, All Years) from:
> https://geodash.vpd.ca/opendata/

Place the CSV file at:
```
data/crimedata_csv_AllNeighbourhoods_AllYears.csv
```

### 3. Train and save the model
This only needs to be done once. It trains all 3 models, picks the best one (Random Forest), and saves it to `model/`:
```bash
python crime_pipeline.py
```

### 4. Install Ollama (for chatbot and GUI only)
Download and install from https://ollama.com/download, then pull the model:
```bash
ollama pull llama3.2
```
> Ollama runs the LLM entirely on your machine — no API key, no internet connection, no cost.

## How It Works

### 1. Data Loading & Cleaning
- Loads the VPD CSV (530,000+ crime records, 2003–present)
- Drops irrelevant columns: `DAY`, `MINUTE`, `HUNDRED_BLOCK`, `X`, `Y`
- Removes rows with missing or blank `NEIGHBOURHOOD` values

### 2. Crime Type Weighting
Each crime type is assigned a severity weight reflecting its relative harm:

| Weight | Crime Types |
|---|---|
| 10 | Homicide |
| 8 | Offence Against a Person, Vehicle Collision (Fatality) |
| 7 | Robbery |
| 6 | Break and Enter Commercial, Break and Enter Residential |
| 5 | Vehicle Collision (Injury), Assault |
| 4 | Theft of Vehicle |
| 3 | Theft from Vehicle |
| 2 | Other Theft, Theft of Bicycle |
| 1 | Mischief |

### 3. Feature Engineering
- Records are grouped by `NEIGHBOURHOOD` and `YEAR`
- Two features are computed per group: `crime_count` and `weighted_score`
- `NEIGHBOURHOOD` is label-encoded to a numeric ID

### 4. Safety Score Formula
The weighted score is min-max normalised and inverted so that higher = safer:
```
safety_score = 100 × (1 - (weighted_score - min) / (max - min))
```

| Score Range | Label |
|---|---|
| 75 – 100 | Safe |
| 50 – 74 | Moderate |
| 25 – 49 | High Crime |
| 0 – 24 | Very High Crime |

### 5. Train / Test Split
A **time-based split** is used to prevent data leakage:
- **Train**: 2003–2019 (408 samples)
- **Test**: 2020–2023 (96 samples)

A random split was deliberately avoided because it would allow the model to see future years during training.

### 6. Models Trained
| Model | MSE | RMSE | R² |
|---|---|---|---|
| Linear Regression | 1.3802 | 1.1748 | 0.9923 |
| Decision Tree | 1.0635 | 1.0313 | 0.9941 |
| **Random Forest** | **1.0042** | **1.0021** | **0.9944** |

**Random Forest** is selected as the best model based on lowest RMSE and highest R². It is saved to `model/best_model.pkl` and loaded by `query.py`, `chatbot.py`, and `gui.py` on startup — no retraining required.

### 7. Future Year Prediction
For years beyond the dataset, the model predicts iteratively:
1. Seeds from the last 3 known years of data
2. Predicts year N+1 using the previous year's score, a 3-year rolling average, and a crime count trend
3. Each predicted year feeds into the next, chaining up to the target year

### 8. Chatbot Architecture
The chatbot uses a two-layer design to avoid hallucination:
- **Layer 1 — Python intent parser**: Extracts neighbourhood names and years from the user's message using regex and string matching. Routes to the correct data tool (stats, compare, ranking, trend, list). This layer never fails or makes things up.
- **Layer 2 — LLM (Ollama / llama3.2)**: Receives the real data fetched by Layer 1 and writes a conversational response. It is explicitly instructed to only use the numbers it was given and never invent facts.
